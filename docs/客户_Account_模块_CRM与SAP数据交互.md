# CRM ↔ SAP 数据交互总览

> 本文档清晰说明：CRM推送什么数据到SAP，SAP同步什么数据到CRM

---

## 三、SAP → CRM（被动同步/查询）

### 接口5: 客户剩余额度查询

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  GetCustomerlimitFROMSAP                                                    │
│  接口端点: getLimit                                                          │
│  接口编号: GSD004                                                           │
└─────────────────────────────────────────────────────────────────────────────┘

触发方式: CRM主动查询SAP，获取客户当前剩余额度

请求内容 (CRM → SAP):
┌─────────────────────────────────────────────────────────────────────────────┐
│ 字段名                     │ 说明                                             │
├───────────────────────────┼─────────────────────────────────────────────────┤
│ partner                   │ 客户编码 (MDG_Customer_Code__c)                   │
│ creditSegment             │ 信用段                                           │
│ WAERS                     │ 币种 (可选)                                      │
│ BUKRS                     │ 公司代码 (可选)                                  │
└───────────────────────────┴─────────────────────────────────────────────────┘

返回内容 (SAP → CRM):
┌─────────────────────────────────────────────────────────────────────────────┐
│ 字段名                     │ CRM字段              │ 说明                    │
├───────────────────────────┼─────────────────────┼──────────────────────────┤
│ partner                   │ -                   │ 客户编码                  │
│ amount                    │ Remaining_Credit__c │ 剩余信用额度             │
│ WAERS                     │ -                   │ 币种                     │
│ zsfyq                     │ Overdue__c          │ 是否逾期 (有值=逾期)      │
│ zovdmtr                   │ -                   │ 逾期金额                 │
│ ZOVDAYS                   │ -                   │ 最大逾期天数             │
│ exch_rate                 │ -                   │ 汇率                    │
└───────────────────────────┴─────────────────────┴──────────────────────────┘

⚠️ 重要: 这个接口是【查询】，不是【推送】
   - 由CRM前端主动调用
   - SAP返回当前剩余额度
   - CRM更新 Remaining_Credit__c 字段
```

## 八、关键业务规则

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                            同步业务规则                                         │
└─────────────────────────────────────────────────────────────────────────────────┘

1. 【客户编码获取】
   ───────────────────────────────────────────────────────────────────────────
   CRM创建客户 → 审批通过 → 同步SAP → SAP返回MDG_Customer_Code__c
   ↑
   这是销售视图同步的前提条件！

2. 【销售视图同步前提】
   ───────────────────────────────────────────────────────────────────────────
   Customer_Sales_Info__c 同步SAP前，必须满足:
   ✓ Account.MDG_Customer_Code__c 不为空
   ✗ 如果为空: 延迟1分钟后通过定时任务同步

3. 【额度同步 vs 额度查询】
   ───────────────────────────────────────────────────────────────────────────
   额度同步 (CRM→SAP):    新建/变更额度时主动推送
   额度查询 (SAP→CRM):    查询当前剩余额度，主动拉取

4. 【订单同步不依赖审批】
   ───────────────────────────────────────────────────────────────────────────
   订单(Order)同步SAP不需要审批流程
   创建/修改订单后直接触发同步

5. 【信用额度变更不触发同步】
   ───────────────────────────────────────────────────────────────────────────
   Remaining_Credit__c 字段变更 不会触发 同步到SAP
   这个字段是由 GetCustomerlimitFROMSAP 从SAP查询回来的
```
